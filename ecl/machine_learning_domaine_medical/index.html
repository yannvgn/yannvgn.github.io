Le Machine Learning dans le domaine médical

Yann Vaginay — @yannvgn

Domaine public. https://unsplash.com/photos/nss2eRzQwgw

Le Machine Learning est omniprésent aujourd’hui. Dans un nombre de domaines croissant, le Machine Learning est employé pour analyser, comprendre, prédire automatiquement ; c’est aussi une discipline qui, depuis relativement peu, suscite la curiosité et l’intérêt de toutes parts de la communauté scientifique.


Son concept n’est pas récent : les développements des théories qui façonnent le Machine Learning remontent à la deuxième moitié du XXème siècle. Après une période timide, c’est avec le développement d’algorithmes performants, l’accroissement des capacités de stockage, de calcul et de parallélisation et l’arrivée du phénomène du Big Data que le Machine Learning a su s’imposer comme un outil indispensable de valorisation des données.

Les applications du Machine Learning sont multiples : moteurs de recherche, finance, vision par ordinateur, météo, marketing, véhicules autonomes, surveillance, industrie.

Bien évidemment, le domaine de la santé n’est pas exclu. Cette étude de veille technologique a pour objectif de donner un aperçu de l’état de l’art et des tendances des utilisations du Machine Learning dans le domaine médical.

Le Machine Learning en bref

“Instead of writing into computer code exactly what to look for, you let the algorithm figure it out.”

Andre Esteva, Thrun lab

Le terme de Machine Learning, ou Apprentissage Automatique, désigne un ensemble de techniques visant à déterminer les paramètres d’un modèle à partir de données dites d’apprentissage. Les données d’apprentissage données en entrée de l’algorithme donnent des prédictions en fonction du modèle. Ces prédictions sont réutilisées et analysées à l’itération suivante pour affiner le modèle (feedback). Itération après itération, on s’attend à ce que le modèle colle plus aux données d’apprentissage.

Pour éviter le problème du sur-apprentissage (que l’on peut comparer à l’apprentissage par coeur, qui s’oppose à la compréhension de l’objet d’apprentissage), un autre jeu de données, appelé données de test, non présenté à l’algorithme lors de l’apprentissage, permet de mesurer la qualité de prédiction.

Il existe différentes mesures de qualité d’un modèle, comme la matrice de confusion ou l’aire sous la courbe ROC. Je vous renvoie à la bibliographie pour plus de précisions. [1]

Les erreurs sont quantifiées par l’intermédiaire d’une fonction de “coût” (quel est le “prix” à payer si l’on se trompe dans une prédiction ?). Une particularité du domaine médical sont des coûts extrêmement élevés en cas d’erreur (on ne peut pas tolérer de faux négatifs – ou du moins le moins possible – à un test de dépistage par exemple).

Catégories d’apprentissage

On distingue deux principales catégories d’apprentissage :

L’apprentissage supervisé. Les données d’apprentissage sont composées, pour chaque individu, de variables explicatives (par exemple les différentes données vitales d’un patient) et d’une variable cible (par exemple malade / non malade ou encore le taux de progression d’une maladie). Il s’agit alors de deviner un moyen de prédire la variable cible en connaissant les variables explicatives. La variable cible peut être catégorielle (dans ce cas, on parlera de classification) ou représenter une valeur numérique continue (il s’agit alors de régression).

L’apprentissage non supervisé. Les données d’apprentissage ne comportent pas de variable cible ; il s’agit là d’extraire / de comprendre la structure des données. Le clustering (regroupement des individus en clusters) et la détection d’anomalies sont des formes d’apprentissage non supervisé.

Principales familles d’algorithmes

Des modèles linéaires aux réseaux de neurones profonds, de nombreux algorithmes de Machine Learning plus ou moins sophistiqués ont été développés. Parmi eux, quelques uns reviennent souvent :

Les réseaux de neurones[2]. Les réseaux de neurones sont revenus à la mode depuis la démocratisation du calcul distribué et de l’augmentation de la capacité de calcul et de stockage. Les réseaux de neurones profonds ont fait leur preuve dans le domaine de la vision par ordinateur entre autres.

Les arbres de décision[3]. À chaque niveau de l’arbre, les données sont scindées selon la valeur d’une variable explicative. Les arbres de décision sont a priori simples à interpréter.

Les réseaux bayésiens[4]. Permettent de déterminer des relations (dépendance, causalité) dans les variables explicatives.

Les classifieurs linéaires[5]. Les classifieurs linéaires sont souvent utilisés en première approche car ils sont simples et donnent des résultats corrects. Une généralisation des classifieurs linéaire sont les SVM (Machines à Vecteurs de Support), intensément utilisés.

Les algorithmes de clustering[6]. Ils permettent un apprentissage non supervisé en formant des clusters de données. Ex : KMeans, DBScan, …

Les données

Les algorithmes ne font pas tout : les données sont un élément essentiel dans le Machine Learning. Les algorithmes de Machine Learning nécessitent généralement un grand volume de données pour apprendre correctement et de la qualité des données dépend la qualité des prédictions : leur collecte et leur préparation sont une tâche qu’il ne faut pas négliger.

Les sources

Le domaine de la santé n’est pas passé outre les révolutions apportées par l’ère du Big Data. Les sources de données sont multiples et leur volume s’accroît.

Les dossiers médicaux comme le DMP (Dossier Médical Partagé) en France ou l’EHR (Electronic Health Record) dans les pays anglophones se généralisent ; ils ont vocation à contenir tout l’historique médical du patient et d’autres données utiles sur le patient. Sa taille pourrait atteindre 6 téraoctets par patient[7].

À ces données s’ajoutent les données récoltés à travers l’utilisation d’Objets Connectés. Avec le développement de nombreuses applications smartphone dédiées à la santé et la mode qui s’en est suivie, l’Internet Of Things constitue une source de données précieuse : certaines données physiologiques peuvent être collectées de façon continue et en temps réel, et ce de manière non intrusive. L’IDC prédit que d’ici 2019, 40% des organismes de santé utiliseront l’IoT pour mesurer de façon plus discrète les signes vitaux des patients[8].

Enfin, de nombreux datasets sont disponibles librement[9], notamment dans le cadre de l’Open Data, et constituent des ressources pour les chercheurs et scientifiques. Par exemple, le site internet data.gov met à disposition de nombreuses données sur la santé publique au États-Unis ; ou encore Kaggle qui propose de nombreuses compétitions de Data Science (notamment dans le domaine de la santé, à l'instar de Data Science Bowl 2017, compétition la plus active actuellement, qui porte sur la détection du cancer du poumon) accompagnées de datasets.

Des données complexes

Mais les données sont aussi le point problématique du domaine médical pour des approches de Machine Learning. En effet, plusieurs aspects des données médicales constituent un frein aux avancées du Machine Learning dans ce domaine.

Les données médicales sont soumises à la subjectivité et aux erreurs des professionnels, mais aussi à la bonne foi du patient ; il en résulte des données bruitées et manquantes. Le manque de standards internationaux et la segmentation des données entre les différentes structures qui les ont obtenues s’ajoutent aux facteurs limitant leur exploitation. Aussi, le grand nombre de paramètres en jeu en font des données complexes[10].

Des données protégées

En outre, les données médicales sont des données sensibles et personnelles : un cadre légal est nécessaire pour leur protection. Cette contrainte s’ajoute aux freins à leur manipulation listés plus haut.
En France, l’ASIP Santé, ou encore Agence Française de la Santé Numérique, s’occupe de promouvoir et d’accompagner les projets de e-santé[11]. Elle pilote la définition du cadre légal concernant le traitement et le stockage de données médicales, en plus de la CNIL. Elle est habilité à décerner des certifications et des habilitations.

“Dans l'esprit de la loi, la volonté des pouvoirs publics est d’organiser le dépôt et la conservation des données de santé dans des conditions de nature à garantir leur pérennité et leur confidentialité, de les mettre à la disposition des personnes autorisées selon des modalités définies par contrat, et de les restituer en fin de contrat.”[12]

Aux États-Unis, c’est l’HIPAA qui fixe au niveau fédéral les réglementations sur ces données.

Le partage des données médicales

La vidéo suivante montre l’importance de la communication des données médicales entre différentes structures.

Différents groupes de travails et conférences s’organisent pour travailler sur ce sujet. Quelques exemples : #datasaveslives, Interop Summit.

Le réel enjeu est la mise à disposition / centralisation des données médicales tout en assurant le respect de la vie privée et la sécurité de ces données. Il n’est pas facile de faire confiance en de tels systèmes d’information pour la santé : chaque année, des données médicales de milliers de patients sont piratées ou fuitent par négligence[13].

Les applications actuelles du Machine Learning pour la santé

Comme annoncé plus haut, le Machine Learning est déjà bien présent dans le monde médical. Lister ici tous les cas d’utilisation du Machine Learning dans le domaine médical serait trop ambitieux ; cependant voici les applications majeures que j’ai repérées au cours de ma veille.

L’aide au diagnostic

L’aide au diagnostic est l’exemple typique de classification supervisée appliquée à la santé. L’idée est de proposer un diagnostic ou une piste de diagnostic à partir de données dont l’obtention est moins coûteuse, plus rapide ou plus facile que le diagnostic classique en lui-même. L’analyse peut par exemple être basée sur des réponses à un questionnaire posé au patient ou sur des mesures de données physiologiques.

  • Aviva remplace certains examens médicaux coûteux par des questionnaires lors d’une souscription à certaines assurances.
  • HealthTap propose aux utilisateurs de son application Dr. AI un diagnostic rapide en fonction des symptômes[14] (là aussi il s’agit de répondre à des questions). L’application permet de rassurer l’utilisateur, de le mettre en contact avec un médecin et de l’alerter si le cas suspecté semble grave (et de l’inviter à se rendre aux urgences).

Ces diagnostics sont rendus possible par un apprentissage sur des cas rencontrés par le passé.

Aide au traitement après diagnostic

Dans ce cas, le médecin a donné son diagnostic et obtient des suggestions de traitement. IBM Watson propose une telle solution[15]. L’apprentissage se fait sur des données de patients ainsi que sur l’analyse de faits et de documents scientifiques. Un argument pour l’emploi de tels systèmes est la détection d’erreurs de traitement / dosage qui peuvent parfois se révéler fatals.

Imagerie médicale

La vision par ordinateur est un des grands centres d’intérêt du Machine Learning. Le développement des réseaux de neurones profonds (Deep Learning) rendu possible par des systèmes de calculs distribués de plus en plus performants a permis d’obtenir des résultats très prometteurs en extraction de caractéristiques d’une image.

Appliqué à l’imagerie médicale, des systèmes de Machine Learning sont capables de détecter des éléments caractéristiques d’une pathologie dans les radiographies ou les IRM.

C’est ainsi que Sebastian Thrun (Stanford Artificial Intelligence Laboratory) et son équipe ont développé une algorithme de Machine Learning capable d’identifier de potentiels mélanomes, responsables du cancer de la peau, à partir d’images de grains de beauté[16]. Cette inspection visuelle, jusqu’alors réalisée par les dermatologistes, est la première étape dans le diagnostic du cancer de la peau ; s’ensuit une biopsie lorsqu’un mélanome est suspecté. L’algorithme s’est avéré aussi performant que des dermatologistes, avec une AUC (aire sous la courbe ROC) d’au moins 91%. L’étape suivante serait le portage de leur procédé sur smartphone pour mettre à disposition du public un moyen rapide et fiable de diagnostic.
La phase de récupération et de préparation des données d’apprentissage a été une étape essentielle dans l’obtention de résultats probants.

Un autre exemple est la startup Zebra Medical Vision[17], spécialisée dans ce domaine.

Prédiction de données quantitatives pour l’administration d’organisations

L’emploi du Machine Learning ne s’arrête pas au patient lui-même, mais peut être étendu aux structures de soins dans une optique d’amélioration de la qualité des services.

C’est l’exemple de quatre hôpitaux de l’AP-HP (Assistance Publique - Hôpitaux de Paris) qui ont mis en place un système de prédiction du taux de fréquentation de l’hôpital[18]. Pour augmenter la qualité des prédictions, des données internes (notamment les fréquentations des dix années antérieures) mais aussi externes (la météo par exemple) ont été utilisées. Le système est capable de prédire les fréquentations à l’heure près. En plus de fournir des prédictions précieuses, le développement de ce système a contribué au projet opensource TAP qu’il utilise en proposant une implémentation distribuée de l’analyse des séries temporelles.

Découverte de nouveaux médicaments

Le Machine Learning se retrouve aussi à la source du traitement des maladies. Aujourd’hui, il est estimé que la mise sur le marché d’un nouveau médicament (depuis sa découverte jusqu’à la commercialisation) prend environ 13 ans et peut coûter plusieurs milliards d’euros[19]. Appliquer le Machine Learning pour analyser des substances avec telle ou telle propriété et en découvrir de nouvelles pourrait accélérer le rythme des découvertes de nouveaux traitements et abaisser drastiquement les coûts de développement (jusqu’à 70% d’après une étude de la Carnegie Mellon University et l’Albert Ludwig University of Freiburg[20])

Des startups comme Benevolent.AI[19] ou Insilico Medicine[21] mais aussi Google[22] effectuent des travaux pour la découverte automatique de médicaments.
Pour découvrir de nouveaux composés susceptibles d’avoir des effets vertueux dans le traitement d’une pathologie, des modèles de deep learning (réseaux de neurones profonds) sont entraînés à partir de composés dont les propriétés sont connues. Après manipulation de la structure du réseau de neurones, l’algorithme est en mesure de proposer des molécules pouvant avoir des propriétés demandées. Dans certains cas, des molécules connues mais non présentées lors de l’apprentissage ont été ainsi retrouvées.

Là encore, la rareté des données est pointée du doigt : près de la moitié des résultats des tests cliniques ne sont pas publiés[19].

Collecte et mise à disposition des données

En raison du manque de données exploitables (pour des raisons données dans la partie précédente), des initiatives de facilitation de collecte des données ont vu le jour. C’est ainsi qu’Apple a conçu les frameworks opensource ResearchKit et CareKit[23]. Le premier est centré sur les professionnels de la santé et destiné à la fabrication d’applications vouées à la réalisation d’études cliniques. Il facilite la mise en relation entre les chercheurs et les utilisateurs désireux de participer à une étude. L’avantage de l’utilisation d’un smartphone (un iPhone dans ce cas) comme interface entre les chercheurs et les participants à l’étude est la facilité de collecter continuellement des données de façon non gênante. Ces études médicales étant à portée de doigt, plus de données sont recueillies, et sur plus de participants qu’avec des méthodes traditionnelles : l’application mPower, développée avec ResearchKit, a permis aux chercheurs de mener une étude sur la maladie de Parkinson basée sur plus de 10000 participants (ce chiffre n’avait jamais été atteint auparavant pour cette maladie).
Le second framework, CareKit, centré sur l’utilisateur, permet de construire plus facilement des applications pour la santé.

Apple a d’ailleurs été placée deuxième dans un classement des entreprises les plus innovantes dans le domaine de la santé pour avoir traité des données pour construire des applications liées à la santé[24].

Perspectives

Au vu des applications du Machine Learning dans le domaine médical et de la sophistication croissante des intelligences artificielles, les espoirs sont grands quant aux avancées qui pourront être faites dans la santé. Pour se rendre compte de la croissance du Machine Learning dans la santé, il n’y a qu’à remarquer la prolifération des startups dans ce domaine précis.

Médecine personnalisée

La médecine personnalisée est une grande attente de l’application du Machine Learning dans les soins. En tenant compte de l’ensemble des données disponibles sur le patient, le choix d’un traitement parmi d’autres pourrait être plus éclairé. Dans le cas du cancer de la prostate, la médecine personnalisée permettrait de retarder de façon sûre le traitement (traitement dont les conséquences sont souvent lourdes dans le quotidien des patients) en surveillant activement la propagation de la maladie[25]. En effet, il est estimé que le cancer de la prostate est surtraité.

Il en suivrait aussi un meilleur accompagnement des patients (suivi après hospitalisation, suivi lors d’un traitement…).

Prédictions plus fiables

Avec l’accroissement des données traitées, on s’attend à des prédictions plus fiables. On peut prendre l’exemple – morbide certes – de la fin de vie : avec une information plus précise sur la fin de vie du patient, le surtraitement et l’acharnement thérapeutique pourront être évités et le médecin sera mieux préparé à accompagner le patient et sa famille[26].

Mutation des métiers

Une chose est sûre, les métiers de la santé et ceux liés aux Data Sciences vont se rapprocher et les professionnels seront amenés à se rapprocher.
Le quotidien des professionnels de santé va aussi évoluer : selon l’IDC, 30% du temps des professionnels de la santé pourrait être libéré par la collecte et l’analyse automatiques des données[8].

Médecine pour tous

Enfin, une grande attente de l’application du Machine Learning à la santé et plus généralement du développement d’applications sérieuses smartphone dédiées à la santé est l’accès rapide, à moindre coût et à tous à la médecine. Des applications smartphones de diagnostic peuvent par exemple être utilisées comme solution – certes limitée – dans des zones où l’accès aux soins est limité.

Limites

Il ne faut tout de même pas perdre de vue les limites de l’application du Machine Learning au domaine médical.

Méfiances des professionnels et des patients

C’est un point inévitable. Accepter et faire confiance à un “raisonnement” fait par une machine n’est pas facile, surtout lorsque la vie de personnes est en jeu.
Les professionnels sont prudents par rapport aux algorithmes pour les raisons suivantes :

  • peur de voir leur métier transformé voire d’être remplacés par des machines ;
  • complexité de leur métier et de leur savoir-faire qui ne pourrait pas être intégrée dans une machine ;
  • importance de l’aspect humain.
Pour les patients, il est naturel de préférer un diagnostic venant d’un médecin – enfin pas tout le monde : Vinod Kholsa, co-fondateur de Sun Microsystems et fondateur de Khosla Ventures, second investisseur dans le domaine de la santé digitale, affirme préférer un bon algorithme de machine learning pour le diagnostic d’une maladie plutôt qu’un médecin moyen. L’utilisation massive de données personnelles vient aussi renforcer leur méfiance. À cela vient s’ajouter une toute autre problématique : veut-on vraiment tout savoir sur sa santé ? par exemple – exemple extrême je le concède – veut-on vraiment connaître le moment de sa mort ?

Modèles en boîte noire

Les modèles développées à l’aide du Machine Learning ont bien souvent la fâcheuse tendance à fonctionner en “boîte noire” : le modèle fournit des prédictions à partir de données d’entrée mais il est impossible de suivre le cheminement logique qui a poussé le système à faire cette prédiction.
Les réseaux de neurones en sont un bon exemple ; même les arbres de décision deviennent abstraits lorsqu’ils deviennent complexes (random forests, boosting…).

Sans possibilité d’interprétation, ces modèles perdent de l’intérêt pour la communauté médicale.

Protection des données

La protection des données reste une problématique majeure de l’application du Machine Learning dans le domaine médical. Je vous renvoie plus haut pour en relire les détails.

Conclusion

Le Machine Learning n’est pas prêt de remplacer les médecins, mais il constitue un formidable outil pour les professionnels de la santé qu’il serait absurde d’ignorer. Le Machine Learning et l’intelligence artificielle ont un bel avenir dans le domaine médical, mais il ne faut surtout pas perdre de vue les problématiques des sujets de son application – patients, médecins, scientifiques.

“I think the biggest innovations of the 21st century will be at the intersection of biology and technology. A new era is beginning.”

Steve Jobs
